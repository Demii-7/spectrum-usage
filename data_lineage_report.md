# Data Lineage Report: Raw SigMF → Ground-Truth Spectrograms

## Step-by-Step Pipeline Comparison

### Step 1: Raw SigMF Archives

**Source:** AERPAW sub-6 GHz dataset, DOI 10.5061/dryad.hmgqnk9zn

| Archive | Sweeps | Time span |
|---------|--------|-----------|
| `ResultsCC1Feb2022_SigMF.zip` | 32,529 | 10,243 unique minutes |
| `ResultsCC2Feb2022_SigMF.zip` | 34,865 | 10,080 unique minutes |
| `ResultsLW1Feb2022_SigMF.zip` | 21,617 | 9,519 unique minutes |

Each sweep = 1 `.sigmf-meta` JSON (metadata) + 1 `.sigmf-data` binary (98,868 × float32 dBm).

Metadata shows: `core:datatype = rf32_le`, frequency span 87.16–6019.18 MHz, center frequency 3053 MHz, data converted from MATLAB.

---

### Step 2: Frequency Sub-Band Selection

| | Our pipeline | Paper / Repo |
|---|---|---|
| **Input** | `--band-start-mhz 87` | 85 MHz (paper) |
| **Width** | `--band-width-mhz 250` | 250 bins |
| **Output range** | 87–336 MHz | 85–335 MHz (paper) |
| **Actual raw data start** | 87.16 MHz | 87.16 MHz |

**Difference:** None significant. Paper says 85–335 but raw data starts at 87.16. Both effectively select the same 250 × 1 MHz bins.

**Method (both):** `build_bin_ranges()` scans the 98,868-bin frequency axis to find which raw bins fall in each 1 MHz interval. ~16–17 raw bins per 1 MHz output bin.

---

### Step 3: Per-Sweep dBm → Linear Conversion

**Both:** Each raw sweep's selected bins are converted from dBm to linear power:
`linear = 10^(dBm/10)`

---

### Step 4: Per-Minute Averaging

**Both:** Multiple sweeps in the same UTC minute are averaged in linear domain, then converted back to dBm:
`dBm_out = 10 * log10(mean(linear_values))`

Single-sweep minutes pass through unchanged (still go through linear conversion).

---

### Step 5: Cross-Node Merge

**Both:** Only minute buckets present in ALL archives are kept (set intersection). Rows sorted chronologically.

**Result:** 6,839 common minutes.

---

### Step 6: CSV Formatting

| | Our pipeline | Repo CSV |
|---|---|---|
| Header | Yes (`CC1_2813`, `CC1_2814`, ...) | **No header** |
| Format | 6 decimal places | 4–5 decimal places |
| Layout | CC1(0–249) → CC2(250–499) → LW1(500–749) | Same assumed |
| Row order | Sorted UTC minutes | Same (same row count) |

## Critical Difference Found

### Statistical fingerprint of the two CSVs

| Metric | Our CSV (87–336) | Repo CSV |
|--------|-------------------|----------|
| **CC1 freq-std per step** | **13.23 dBm** | **0.85 dBm** |
| **CC2 freq-std per step** | **11.24 dBm** | **1.17 dBm** |
| **LW1 freq-std per step** | **11.46 dBm** | **1.72 dBm** |
| CC1 temporal variation | 0.86 dBm | 2.29 dBm |
| CC2 temporal variation | 1.50 dBm | 4.55 dBm |
| LW1 temporal variation | 0.76 dBm | 0.08 dBm |
| CC1 freq range across bins | 49.17 dBm | 3.77 dBm |
| CC2 freq range across bins | 52.04 dBm | 4.21 dBm |
| LW1 freq range across bins | 59.62 dBm | 17.66 dBm |

### Interpretation

**Our CSV** shows a realistic spectrum profile:
- Strong frequency selectivity (~11–13 dBm std across bins) — different frequencies carry different signals
- Weak temporal variation (~1 dBm per bin over time) — the RF environment is stable at fixed locations
- Range of ~50–60 dBm across the band — realistic for VHF/UHF (strong broadcast signals against noise floor)

**Repo CSV** shows an implausible profile:
- Near-zero frequency selectivity (0.85–1.72 dBm std across bins) — all 250 bins per node have nearly the same power
- Slightly higher temporal variation than ours — suggesting a single aggregate value per time step rather than 250 independent bins
- LW1 is essentially constant: mean=−133.97 dBm, std=0.08 dBm over time, 1.72 dBm across frequency

### Root Cause Hypothesis

The repo CSV was **not generated by `build_training_csv.py`** or any equivalent raw-SigMF-to-CSV pipeline. The statistical fingerprint suggests:

1. **The 750 columns do NOT represent 250 independent frequency bins per node.** They may be duplicates of a single aggregated value per node, or heavily averaged across frequency.

2. **LW1 is a constant vector** (temporal variation 0.08 dBm) — essentially the noise floor with no signal content.

3. The repo CSV appears to be the output of a **different, undocumented processing pipeline** (possibly MATLAB-based, since the SigMF metadata says "converted from MATLAB").

### Conclusion

Our `build_training_csv.py` pipeline is **not the source of the disagreement**. The repo CSV on the `Demii-7/TSS-LCD` `main` branch was produced by an entirely different, undocumented process. Our pipeline correctly converts raw SigMF to per-minute averaged CSV with realistic spectrum statistics.
