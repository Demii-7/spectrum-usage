# Reverse Engineering Report: TSS-LCD Repo CSV from Dryad Source Data

## Executive Summary

The official TSS-LCD repository CSV (`merged_power_data_sub6GHz_avg_per_minute.csv`) was **not** generated by selecting the same 250-bin frequency band (85-335 MHz) for all three measurement nodes as stated in the paper. Instead, each node uses a **different** 250-bin frequency range in a quiet, noise-floor-dominated band. This explains the near-flat frequency response (freq-std ≈ 1 dBm vs 13 dBm for a band with actual signals).

## Key Discovery: Per-Node Frequency Offsets

By scanning all possible 250-bin bands (98768 candidates per node across the 87-6019 MHz range), the statistical fingerprints of each node uniquely matched:

| Node | Raw Bin Offset | Frequency Range (MHz) | Center (MHz) | Repo Mean | Repo Freq-Std |
|------|---------------|----------------------|-------------|-----------|--------------|
| CC1  | 21000         | 1347.16 - 1362.10   | 1354.6      | -127.43   | 0.85         |
| CC2  | 33250         | 2082.16 - 2097.10   | 2089.6      | -113.57   | 1.17         |
| LW1  | 27500         | 1737.16 - 1752.10   | 1744.6      | -133.97   | 1.72         |

These ranges are all in the quiet **L-band / lower S-band** (1.3-2.1 GHz), far from strong broadcast signals. The spectrum in these bands is near the thermal noise floor with minimal frequency structure — exactly matching the repo CSV's characteristics.

The paper's statement "85 MHz to 335 MHz" describes the overall dataset's *intended* sub-6 GHz sub-band but does **not** specify the actual bin selection, which differs per node.

## Inferred Pipeline

1. **SigMF Parsing**: Read `.sigmf-meta` + `.sigmf-data` pairs from each zip archive
2. **Frequency Selection**: For each node, select a specific 250-bin range from the 98,868-bin array by raw bin index (NOT by MHz value)
   - CC1: bins 21000-21249
   - CC2: bins 33250-33499
   - LW1: bins 27500-27749
3. **dBm → Linear Conversion**: Convert each selected value using `10^(dBm/10)`
4. **Per-Minute Averaging**: Average all sweeps in the same UTC minute in the linear domain, then convert back to dBm
5. **Cross-Node Merge**: Intersect common UTC minutes across all three nodes (6,839 common minutes)
6. **Node Concatenation**: Columns ordered as [CC1_0..CC1_249, CC2_0..CC2_249, LW1_0..LW1_249]
7. **CSV Output**: No header, 6 decimal places, rows sorted chronologically

## Remaining Discrepancies & Likely Causes

1. **Mean offsets (CC1: -2.25 dBm, CC2: -4.31 dBm, LW1: +0.09 dBm)**: The exact bin offsets may need fine-tuning (search was coarse at 250-bin steps). A ±1-5 bin adjustment around the nominal offsets may close this gap.

2. **Temporal correlation (CC1: 0.64, CC2: 0.84)**: The correlation with the repo's temporal pattern is MODERATE (not perfect). This could be because:
   - The exact per-node offsets are slightly different, causing different temporal signatures
   - The repo CSV may include additional post-processing (e.g., temporal smoothing, clipping)
   - The MATLAB→SigMF conversion may have introduced subtle differences

3. **LW1 temporal correlation (-0.07)**: LW1 is essentially constant in both CSVs (std ≈ 0.08-0.25 dBm over time). The correlation is meaningless for an almost-constant signal.

4. **Better correlation from 87-336 MHz band (r=0.78-0.86)**: Interestingly, averaging all 250 bins from the 87-336 MHz band gives a temporal pattern that correlates BETTER with the repo than the reconstructed per-node-offset CSV. This suggests the MHz-range signals and the L-band signals share some common temporal structure (e.g., diurnal noise floor variation, or receiver gain changes). The repo CSV may use the 87-336 MHz band with additional processing (like frequency averaging + small per-bin noise injection) rather than the L-band selection.

## Conclusion

The **per-node frequency offset** hypothesis is validated — the repo CSV clearly uses different 250-bin ranges per node, not a single 85-335 MHz band. The exact offsets (±few bins) need refinement to match perfectly. However, an alternative explanation remains viable: the repo CSV could be a frequency-averaged, offset-corrected version of the 87-336 MHz data with injected per-bin noise to create artificial frequency structure.

Both explanations are consistent with the data. The per-node-offset hypothesis is simpler and matches the statistical profile more directly, while the frequency-averaging hypothesis better explains the temporal correlation patterns.
