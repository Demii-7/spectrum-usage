8. Errors, Failures, and Inconveniences Encountered

   [E-1] CUDA driver too old (SYSTEM LIMITATION)
         Symptom: "CUDA initialization: The NVIDIA driver on your system is
         too old (found version 12060)."
         Root cause: NVIDIA driver 12060 < 12070 required by PyTorch 2.12.1.
         Impact: All training runs on CPU (~1-5s/batch).
         Workaround: export CUDA_VISIBLE_DEVICES="". No script fix possible.

   [E-2] np.loadtxt fails on CSV with commas (BUG - FIXED)
         Symptom: "could not convert string to float32"
         Root cause: np.loadtxt defaults to whitespace delimiter, not comma.
         Fix: Replaced np.loadtxt with pd.read_csv(header=None).values.

   [E-3] NEN skip-connection shape mismatch (BUG - FIXED)
         Symptom: "RuntimeError: Sizes of tensors must match except dimension 1"
         Root cause: The original EnhancedNoiseNet saved encoder outputs after
         each MaxPool, but skip connections require pre-pool outputs. Generalized
         encoder/decoder into _EncBlock/_DecBlock classes with explicit
         pre_pool outputs.
         Fix: Restructured NEN with separate EncBlock/DecBlock submodules.

   [E-4] LSD output shape mismatch with dataset (BUG - FIXED)
         Symptom: LSE expected (B, T, L, F) but dataset returns (B, T, L*F).
         Root cause: LSE/LSD used 4D (B, T, L, F) internally but dataset and
         loss computation use flat (B, T, L*F).
         Fix: LSE accepts flat input, LSD outputs flat. Reshape happens inside
         model components only as needed (Conv2D/TSS-CC branches).

   [E-5] TSSConditionConstructor forward mismatched FFM signature (BUG - FIXED)
         Symptom: "FeatureFusionModule.forward() missing 1 required positional
         argument: 'H_spatial'"
         Root cause: FFM originally had 3-arg signature (H_temporal, H_spectral,
         H_spatial), but TSS-CC forward called it with pre-combined Q and KV.
         Fix: Simplified FFM to 2-arg (x_q, x_kv); TSS-CC handles concatenation.

9. Issues and Notes
   - Negative R² is expected: 588K parameter model on 138 training windows
     (2 hours of single-node data) lacks sufficient signal.
   - Full 3-node dataset (6839 rows) with paper-default config should produce
     meaningful results.
   - Diffusion noise loss (Stage 3) did not decrease significantly in 2 epochs;
     full training with 1000 epochs is needed for convergence.
   - The projection_to_latent bridge converges quickly (Stage 2 val loss 0.01)
     even with small model and data.
   - All evaluation metrics computed on denormalized dBm values.
   - Plots use constrained_layout, Viridis spectrogram, RdBu_r error colormap.
